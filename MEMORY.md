# Проект: Сайт-портфолио ilya_pi

## Обзор
Сайт-портфолио для специалиста в дизайне, маркетинге, создании сайтов и digital-технологиях.
Чистый HTML/CSS/JS, без фреймворков. Расположение: `s:\PYTHON\ilya_pi\`

## Ключевые механики

### 1. Шапка и бургер-меню
- **На hero**: полная шапка — лого (SVG домик) + навигация + кнопка "обсудить проект"
- **После скролла ниже hero**: шапка скрывается (`header--hidden`), бургер-кнопка появляется (`burger--visible`)
- **На <=992px**: навигация и CTA скрыты через CSS, бургер виден всегда
- **На cases.html и case-detail.html**: нет hero → бургер сразу видимый
- **Бургер**: выезжает панель справа (`menu-panel--active`), overlay, кнопка "Обсудить проект" внизу
- **Закрытие**: крестик, overlay, Escape, клик по ссылке

### 2. Кнопка "обсудить проект" (header)
- Белый бордер (`header__cta-text`)
- Красный псевдоэлемент (`header__cta-bg`) смещён top:-10px, left:-18px, width:70%
- На hover за **1s** `header__cta-bg` плавно входит внутрь кнопки (top:0, left:0, width:100%)

### 3. Система кейсов
- Данные в `data/cases-data.js` — массив `CASES_DATA`
- Каждый кейс: `{ id, title, category, tags[], image, images[], description, fullDescription: {task, solution, result} }`
- Категории: `design`, `marketing`, `web`, `digital` (+ любые новые)
- Категории для фильтра извлекаются автоматически из данных
- На главной: первые 6 кейсов + кнопка "Посмотреть все"
- На cases.html: все кейсы + фильтр кнопками по категориям
- Клик по кейсу → `case-detail.html?id=N`

### 4. Детальная страница кейса
- JS читает `?id=` из URL, находит кейс в `CASES_DATA`
- Рендерит: заголовок, категория, теги, главное изображение, 3 блока (задача/решение/результат), галерея
- Если кейс не найден — страница "Кейс не найден"

### 5. Секция "Направления" (Услуги)
- **Layout**: 2 колонки — слева аккордеон, справа 3D viewer (500px высота)
- **Аккордеон**: 4 направления (Дизайн, Маркетинг, Создание сайтов, Digital-решения)
  - Переключение по клику
  - Автопереключение каждые 5 секунд (таймер с прогресс-баром)
  - Пауза таймера при наведении на аккордеон
- **3D контент**: Уникальная сцена для каждого направления
  - Дизайн: вращающийся куб + кольцо + точка
  - Маркетинг: пирамида + кольцо + точка
  - Создание сайтов: сфера (пульсация) + орбита + точка
  - Digital: октаэдр + кольцо + точка
- **Паралакс**: 3D объекты реагируют на движение курсора в блоке viewer
- **Анимации**: вращение, парение (float), пульсация
- **Адаптив**: <992px — 1 колонка, 3D блок ниже аккордеона (350px)

## Анимации и эффекты

### Hero
1. **Particle Network** (canvas, `#heroCanvas`) — 80 частиц (40 на мобильных) с линиями-связями. Курсор притягивает линии и отталкивает частицы. Паузится когда hero не виден (IntersectionObserver).
2. **Floating Glow Orbs** (3 div'а `.hero__glow--1/2/3`) — красные/белые размытые сферы, парят по разным траекториям.
3. **Title Mouse Tilt** — заголовок "МАРКЕТИНГ" наклоняется вслед за курсором (3D perspective).
4. **Hero Entrance** — заголовок влетает снизу с blur (1.2s), нижняя панель появляется (задержка 1s), теги по очереди.

### Services
- **Grid Shift** (`.services__bg-grid`) — сетка линий 60x60px медленно ползёт по диагонали (20s linear infinite).

### Contact
- **Aurora Pulse** (`.contact__aurora`) — два радиальных градиента красного плавно меняют позицию/размер (10s).

### Scroll Reveal
- `.reveal` — fade-in + translateY(40px) при скролле (IntersectionObserver, threshold 0.15)
- `.reveal-stagger` — дети появляются по очереди (задержка +0.05s на каждый элемент)
- Применяется к: заголовкам секций, текстам, карточкам услуг, сетке кейсов, CTA

### Reduced Motion
- `@media (prefers-reduced-motion: reduce)` — все анимации отключаются, canvas скрывается

## Адаптивность (breakpoints)

| Breakpoint | Что меняется |
|-----------|-------------|
| **1200px** | Карточки услуг: 4 → 2 колонки |
| **992px** | Навигация/CTA скрыты, бургер всегда видим. About: 1 колонка. Кейсы: 2 колонки. Footer: вертикальный |
| **768px** | Hero title уменьшается, bottom-секция вертикальная. Услуги: 1 колонка. Теги мельче. Галерея: 1 колонка |
| **576px** | Hero 100svh. Кейсы: 1 колонка. Кнопки: меньше padding. Menu panel: 100% ширины |

## Как расширять кейсы

1. Открыть `data/cases-data.js`
2. Скопировать любой объект в массиве
3. Изменить `id` (уникальный), `title`, `category`, `tags`, `description`, `fullDescription`
4. Новые категории подхватываются фильтром автоматически
5. Для изображений: положить файлы в `img/`, указать путь в `image` и `images[]`

## Дизайн-решения

- Макет hero из `Frame 9.svg` (Figma export)
- Логотип: SVG домик (`#AF2F2F` + белое окно)
- Hero заголовок: гигантский condensed italic Oswald (scaleY для вытянутости)
- Теги в hero: outlined pills (белый бордер, прозрачный фон)
- CTA "Хочу тоже": красная скруглённая кнопка, внизу справа hero
- "Обсудить проект": анимированный псевдоэлемент (красный → входит в белую рамку за 1s)
